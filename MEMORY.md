## Architecture (Read details in ARCHITECTURAL_DIGEST.md)
- **Type**: Static site generator (Node.js)
- **Build**: Fetch data → Build HTML → Commit to main → GitHub Pages deploys
- **Source**: `tools/` (build logic, templates, fetch scripts), `shared/` (client modules), `workers/` (Cloudflare)
- **Output**: `archive/` (static HTML), `data/` (raw JSON/cache) — both read-only; do not edit directly
- **Templates**: `tools/lib/templates.js` — generates all HTML (incidents, profiles, weather, webcams, ground conditions)
- **Build orchestrator**: `tools/build.js`

## Data Flow
Sources → Fetch scripts → `data/` → Build → `archive/` → GitHub Pages

**Sources**:
- Lawis.at (bulletins PDF/JSON, incidents, profiles, images)
- LWD Bayern, GeoSphere Austria
- Local weather stations (telemetry)
- Cloudflare KV (user uploads: ground conditions, GPX files)

**Fetch scripts** (in `tools/`):
- `fetch_daily.js` — orchestrates daily updates, downloads bulletins
- `fetch_weather.js` — raw station telemetry
- `fetch_weather_report.js` — daily text weather report (Puppeteer, with retry logic)
- `fetch_lawis_incidents.js` — scrapes incident reports + images
- `fetch_uploads.js` — retrieves user submissions from KV
- `process_profiles.js` — snow profile data
- `enrich_profiles.js` — translates German→English (GCP API), parses stability tests
- `pdf_generator.js` — generates PDFs locally for Tyrol (AT-07) bulletins (used by `pdf_fetcher.js`)

## Key Commands
- `npm ci` — install dependencies
- `npm run fetch:all` — run all fetch scripts
- `npm run build` — generate static site
- `npm run serve` — local preview at http://localhost:3000
- `git pull` / `git push` — sync and deploy

## Automation
- **GitHub Actions**: `.github/workflows/daily-fetch.yml`
- **Schedule**: 3× daily (06:00, 14:00, 18:00 CET) → UTC 05:00, 13:00, 17:00
- **Steps**: checkout → setup Node 20 → `npm ci` → `npm run fetch:all` → `npm run build` → auto-commit (message: "Routine data update") to main
- **Triggers**: commit to main → GitHub Pages deploy
- **Secrets**: `GCP_TRANSLATE_KEY` (Google Cloud API key for translation)

## Cloudflare Workers
- **Config**: `wrangler.toml`
- **Upload API**: `workers/upload-worker.js` — handles user uploads and GPX library
- **KV**: Stores `uploads.json` and GPX metadata/files (`gpx:index`, `gpx:file:<id>`)
- **Deploy**: `wrangler deploy` (run manually when `workers/` changes)

## Important Rules
- Do NOT edit files in `archive/` or `data/` directly — they are overwritten by fetch/build
- For any UI/HTML changes, edit `tools/lib/templates.js` or source in `tools/src/`
- `styles.css` is the global stylesheet
- Always `git pull` before making changes
- Keep documentation (README.md, ARCHITECTURAL_DIGEST.md, section CONTEXT.md) in sync with code changes
- Do not push secrets to repo; use GitHub Actions secrets and .env for local dev

## Common Pitfalls
- Asking where incident HTML files live — they don't exist statically; they are generated by `templates.js`
- Editing built files in `archive/` — will be lost on next build
- Forgetting to update docs after code changes
- Changing cron times in GitHub Actions without understanding CET/UTC offset
